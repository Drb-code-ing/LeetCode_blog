# LeetCode 122. 买卖股票的最佳时机 II —— 递归 → DP → 贪心，三招拆解股票交易

## 前言

"买卖股票的最佳时机 II"是 LeetCode 股票系列中最能体现**一题多解**的题目。它没有交易次数限制，可以多次买卖，这给了我们从不同角度切入的空间：

- **递归**：自顶向下，直观模拟"每天做决策"的思维过程
- **动态规划**：自底向上，用状态机描述"持有/不持有"的转移
- **贪心**：利用"不限次数"的特性，抓住每一段上涨

更重要的是，它和"买卖股票 I"的区别非常微妙——I 只能买卖一次，II 可以多次——理解这种区别，整套股票系列题（共 6 道）就打通了一半。

本文将用三种方法彻底讲透这道题，每种方法都附带 JavaScript 和 Python 双语言实现，以及详细的手动推演过程。

## 问题描述

### LeetCode 122. Best Time to Buy and Sell Stock II（买卖股票的最佳时机 II）

> 给你一个整数数组 `prices`，其中 `prices[i]` 表示某支股票第 `i` 天的价格。
>
> 在每一天，你可以决定是否购买和/或出售股票。你在**任何时候最多只能持有一股**股票。你也可以先购买，然后在**同一天**出售。
>
> 返回你能获得的**最大利润**。

**示例：**

```
输入：prices = [7,1,5,3,6,4]
输出：7
解释：
  第2天买入（价格=1），第3天卖出（价格=5），利润=4
  第4天买入（价格=3），第5天卖出（价格=6），利润=3
  总利润=4+3=7
```

```
输入：prices = [1,2,3,4,5]
输出：4
解释：
  第1天买入（价格=1），第5天卖出（价格=5），利润=4
  注意你不能在第1天和第2天连续买入，必须先卖出再买入。
```

```
输入：prices = [7,6,4,3,1]
输出：0
解释：价格一直下跌，不买不卖就是最优解。
```

## 核心思想

### 这道题和"买卖股票 I"的区别在哪？

- **I（LeetCode 121）**：只能买卖**一次**，需要找到最低买入点和最高卖出点
- **II（本题）**：可以买卖**无数次**，只要每次买卖有利润就可以做

这意味着 II 的答案一定 **≥** I 的答案——因为 I 的最优策略是 II 的一个可行策略。

### 交易的本质

每一天，你面临两种状态：

```
持有股票（hold）：手里有股票，等待高价卖出
不持有股票（free）：手里没有股票，等待低价买入
```

这两个状态之间的转移，就是整个问题的核心：

```
        买入（-prices[i]）
  free ──────────────────→ hold
    ↑                        │
    │                        │
    └──────────────────────┘
        卖出（+prices[i]）
```

一旦理解了这张图，递归、DP、贪心都只是同一种思想的不同实现方式。

## 思路分析

### 解法一：递归（自顶向下 DFS）

每天你可以做三个选择之一：**休息**、**买入**（如果空仓）、**卖出**（如果持仓）。

这天然适合递归：从第 `i` 天开始，给定当前的持仓状态，**后续能获得的最大利润**是多少？

**递归函数定义：**

```
dfs(i, hold) = 从第 i 天开始，当前持仓状态为 hold 时，
            后续能获得的最大利润

hold = 0 表示空仓，hold = 1 表示持仓
```

**状态转移：**

```
dfs(i, 0) = max(
    dfs(i+1, 0),           // 继续空仓（休息）
    dfs(i+1, 1) - prices[i]  // 今天买入，变为持仓
)

dfs(i, 1) = max(
    dfs(i+1, 1),           // 继续持仓（休息）
    dfs(i+1, 0) + prices[i]  // 今天卖出，变为空仓
)
```

**递归边界：** `i === n` 时，没有更多天数，利润为 0。

这个递归树的大小是 O(2^n)，但我们可以用**记忆化**把它优化到 O(n)。

### 解法二：动态规划（自底向上状态机）

递归+记忆化天然可以转化为 DP。定义：

```
dp[i][0] = 第 i 天结束时，空仓状态下的最大利润
dp[i][1] = 第 i 天结束时，持仓状态下的最大利润
```

**转移方程：**

```
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])  // 休息 or 卖出
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])  // 休息 or 买入
```

最终答案：`dp[n-1][0]`（最后一天空仓一定 ≥ 最后一天持仓）

**空间优化：** 每一天的状态只依赖前一天，所以可以用两个变量代替整个数组：

```
ans  = max(ans,  curr + prices[i])   // 空仓状态：休息 or 卖出
curr = max(curr, ans - prices[i])    // 持仓状态：休息 or 买入
```

这就是 DP 状态机压缩到两个变量后的经典形式。

### 解法三：贪心

由于没有交易次数限制，我们可以贪心到极致：

> **抓住每一段上涨！** 只要明天的价格比今天高，今天就买入，明天就卖出。

```
prices = [1, 2, 3, 4, 5]
贪心：1→2(+1), 2→3(+1), 3→4(+1), 4→5(+1) = 4
等价：1→5(+4) = 4
```

两种策略结果相同，但贪心的实现极度简洁。

**为什么贪心在这里成立？**

假设 prices = [a, b, c]，其中 a < b < c：
- 一次买卖：c - a
- 多次买卖：(b - a) + (c - b) = c - a

**结果一样！** 所以只要相邻两天有利润，我们就做——最终利润就是所有正差值的和。

## 代码实现

### JavaScript 版本

#### 方法一：递归 + 记忆化（DFS）

```javascript
/**
 * @param {number[]} prices
 * @return {number}
 */
var maxProfit = function(prices) {
    const n = prices.length;
    const memo = Array.from({ length: n }, () => [-1, -1]);

    /**
     * dfs(i, hold) = 从第 i 天开始，hold 状态下的最大利润
     * hold: 0 = 空仓, 1 = 持仓
     */
    function dfs(i, hold) {
        if (i >= n) return 0;
        if (memo[i][hold] !== -1) return memo[i][hold];

        if (hold === 0) {
            // 空仓：休息 或 今天买入
            memo[i][hold] = Math.max(
                dfs(i + 1, 0),              // 继续休息
                dfs(i + 1, 1) - prices[i]   // 买入股票
            );
        } else {
            // 持仓：休息 或 今天卖出
            memo[i][hold] = Math.max(
                dfs(i + 1, 1),              // 继续持有
                dfs(i + 1, 0) + prices[i]   // 卖出股票
            );
        }
        return memo[i][hold];
    }

    return dfs(0, 0);
};
```

#### 方法二：动态规划（二维 DP）

```javascript
var maxProfit = function(prices) {
    const n = prices.length;
    // dp[i][0] = 第 i 天空仓的最大利润
    // dp[i][1] = 第 i 天持仓的最大利润
    const dp = Array.from({ length: n }, () => [0, 0]);

    dp[0][0] = 0;                    // 第0天空仓，利润0
    dp[0][1] = -prices[0];           // 第0天买入，利润为负

    for (let i = 1; i < n; i++) {
        dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1] + prices[i]);
        dp[i][1] = Math.max(dp[i - 1][1], dp[i - 1][0] - prices[i]);
    }

    return dp[n - 1][0];  // 最后一天空仓
};
```

#### 方法三：DP 空间优化版

```javascript
var maxProfit = function(prices) {
    let ans = 0;                   // 空仓状态的最大利润
    let curr = -Infinity;          // 持仓状态的最大利润（-Infinity 表示不可能）

    for (let i = 0; i < prices.length; i++) {
        // 要么保持空仓，要么卖出（变为空仓）
        ans = Math.max(ans, curr + prices[i]);
        // 要么保持持仓，要么买入（变为持仓）
        curr = Math.max(curr, ans - prices[i]);
    }

    return ans;
};
```

#### 方法四：贪心

```javascript
var maxProfit = function(prices) {
    let profit = 0;

    for (let i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            profit += prices[i] - prices[i - 1];
        }
    }

    return profit;
};
```

### Python 版本

#### 方法一：递归 + 记忆化（DFS）

```python
def max_profit(prices: list[int]) -> int:
    """方法一：递归 + 记忆化"""
    n = len(prices)
    memo = [[-1, -1] for _ in range(n)]

    def dfs(i: int, hold: int) -> int:
        """从第 i 天开始，hold 状态下的最大利润"""
        if i >= n:
            return 0
        if memo[i][hold] != -1:
            return memo[i][hold]

        if hold == 0:
            # 空仓：休息 或 买入
            memo[i][hold] = max(
                dfs(i + 1, 0),              # 继续休息
                dfs(i + 1, 1) - prices[i]   # 买入股票
            )
        else:
            # 持仓：休息 或 卖出
            memo[i][hold] = max(
                dfs(i + 1, 1),              # 继续持有
                dfs(i + 1, 0) + prices[i]   # 卖出股票
            )

        return memo[i][hold]

    return dfs(0, 0)
```

#### 方法二：动态规划（二维 DP）

```python
def max_profit(prices: list[int]) -> int:
    """方法二：动态规划（二维 DP 状态机）"""
    n = len(prices)
    # dp[i][0] = 第 i 天空仓的最大利润
    # dp[i][1] = 第 i 天持仓的最大利润
    dp = [[0, 0] for _ in range(n)]

    dp[0][0] = 0                  # 第0天空仓，利润0
    dp[0][1] = -prices[0]         # 第0天买入，利润为负

    for i in range(1, n):
        dp[i][0] = max(dp[i - 1][0], dp[i - 1][1] + prices[i])
        dp[i][1] = max(dp[i - 1][1], dp[i - 1][0] - prices[i])

    return dp[n - 1][0]  # 最后一天空仓
```

#### 方法三：DP 空间优化版

```python
def max_profit(prices: list[int]) -> int:
    """方法三：DP 空间优化版（两个变量滚动）"""
    ans = 0                 # 空仓状态的最大利润
    curr = float('-inf')    # 持仓状态的最大利润

    for price in prices:
        # 要么保持空仓，要么卖出（变为空仓）
        ans = max(ans, curr + price)
        # 要么保持持仓，要么买入（变为持仓）
        curr = max(curr, ans - price)

    return ans
```

#### 方法四：贪心

```python
def max_profit(prices: list[int]) -> int:
    """方法四：贪心"""
    profit = 0

    for i in range(1, len(prices)):
        if prices[i] > prices[i - 1]:
            profit += prices[i] - prices[i - 1]

    return profit
```

### 逐步推演

以 `prices = [7, 1, 5, 3, 6, 4]` 为例，用四种方法推演。

#### 递归树（部分剪枝）

```
                          dfs(0, 0)
                    /                  \
              休息 dfs(1,0)        买入 dfs(1,1)
              (跳过....)             /         \
                              休息 dfs(2,1)   卖出 dfs(2,0)
                                           (price[1]=1买, price[2]=5卖 → +4)
```

记忆化确保了每个 `(i, hold)` 状态只计算一次，复杂度 O(n)。

#### DP 状态机递推

```
i=0: price=7
  dp[0][0] = 0
  dp[0][1] = -7 (买入)

i=1: price=1
  dp[1][0] = max(dp[0][0]=0, dp[0][1]+1=-6) = 0        (空仓：休息)
  dp[1][1] = max(dp[0][1]=-7, dp[0][0]-1=-1) = -1      (持仓：在价格1买入)

i=2: price=5
  dp[2][0] = max(dp[1][0]=0, dp[1][1]+5=4) = 4          (空仓：卖出！利润=4)
  dp[2][1] = max(dp[1][1]=-1, dp[1][0]-5=-5) = -1       (持仓：保持)

i=3: price=3
  dp[3][0] = max(dp[2][0]=4, dp[2][1]+3=2) = 4          (空仓：休息)
  dp[3][1] = max(dp[2][1]=-1, dp[2][0]-3=1) = 1         (持仓：在价格3买入)

i=4: price=6
  dp[4][0] = max(dp[3][0]=4, dp[3][1]+6=7) = 7          (空仓：卖出！利润=7)
  dp[4][1] = max(dp[3][1]=1, dp[3][0]-6=-2) = 1         (持仓：保持)

i=5: price=4
  dp[5][0] = max(dp[4][0]=7, dp[4][1]+4=5) = 7          (空仓：休息)
  dp[5][1] = max(dp[5][1]=1, dp[4][0]-4=3) = 3          (持仓：买入？)

最终答案: dp[5][0] = 7 ✅
```

#### DP 空间优化版递推

```
初始: ans=0, curr=-∞

i=0, price=7:
  ans  = max(0,    -∞+7=-∞) = 0        (空仓，保持)
  curr = max(-∞,   0-7=-7)  = -7       (在价格7买入)

i=1, price=1:
  ans  = max(0,    -7+1=-6) = 0        (空仓，不卖)
  curr = max(-7,   0-1=-1)  = -1       (更新买入价到1)

i=2, price=5:
  ans  = max(0,    -1+5=4)  = 4        (以价格1买，5卖 → 利润4)
  curr = max(-1,   4-5=-1)  = -1       (持仓不变)

i=3, price=3:
  ans  = max(4,    -1+3=2)  = 4        (空仓，不卖)
  curr = max(-1,   4-3=1)   = 1        (在价格3买入)

i=4, price=6:
  ans  = max(4,    1+6=7)   = 7        (以价格3买，6卖 → 利润3，总=7)
  curr = max(1,    7-6=1)   = 1        (持仓不变)

i=5, price=4:
  ans  = max(7,    1+4=5)   = 7        (空仓，不卖)
  curr = max(1,    7-4=3)   = 3

最终: ans = 7 ✅
```

#### 贪心推演

```
prices = [7, 1, 5, 3, 6, 4]

i=1: 1 > 7 ? → 否
i=2: 5 > 1 ? → 是 → profit += 5-1 = 4  (1买5卖)
i=3: 3 > 5 ? → 否
i=4: 6 > 3 ? → 是 → profit += 6-3 = 3  (3买6卖)
i=5: 4 > 6 ? → 否

总利润 = 4 + 3 = 7 ✅
```

## 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| **递归+记忆化** | O(n) | O(n) | 最直观，体现问题结构 | 递归栈开销 |
| **二维 DP** | O(n) | O(n) | 状态转移清晰 | 空间 O(n) |
| **DP 空间优化** | O(n) | O(1) | 空间最优的 DP | 不够直观 |
| **贪心** | O(n) | O(1) | 最简洁 | 仅适用于不限交易次数 |

## 优化方向

### 1. 从 DP 到贪心的本质

当没有交易次数限制时，DP 中的状态机没有容量约束，贪心就成了 DP 的极限简化：

```
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])
```

如果把"买入"和"卖出"看作可以在相邻两天完成的操作，那么每次 `prices[i] > prices[i-1]` 的上涨都可以贪心吃掉。这是**无限交易次数**这个条件赋予的优势。

### 2. 为什么 `curr` 初始化为 `-Infinity`？

`curr` 表示"持有股票时的最大利润"。第一天之前不可能已经持有股票，所以用 `-Infinity` 表示一个不可能的坏值。如果用 `0` 初始化会出错——它会凭空持有一股 0 成本的股票。

```
第0天 price=7:
  curr = -Infinity → curr = max(-Infinity, 0-7) = -7 ✓  正确
  curr = 0         → curr = max(0, 0-7) = 0      ✗  错误（不可能持有）
```

### 3. 扩展到有交易次数限制

这题的框架可以无缝扩展到"买卖股票 III"（最多 2 次交易）和"IV"（最多 k 次）：

```
// II（本题，无限次）：2 个状态
dp[i][0], dp[i][1]

// III（最多 2 次）：4 个状态 × k 次
dp[i][k][0], dp[i][k][1]

// IV（最多 k 次）：2k 个状态
dp[i][k][0], dp[i][k][1]
```

## 举一反三

LeetCode 股票系列六道题，核心都是同一个状态机，区别只在于"最多可以交易几次"：

| 题目 | 交易次数 | 特点 |
|------|---------|------|
| **121. I** | 1 次 | 一次买一次卖，找最大差值 |
| **122. II**（本题） | 无限 | 抓住所有正差值，贪心或 DP |
| **123. III** | 2 次 | DP 加一维记录交易次数 |
| **188. IV** | k 次 | III 的泛化，注意 k 很大时退化 |
| **309. 冷冻期** | 无限但有冷却 | 卖出后等一天才能买入，DP 加一维 |
| **714. 含手续费** | 无限但有费用 | 每次卖出（或买入）扣除手续费 |

通用状态机模板：

```
dp[i][k][0] = max(dp[i-1][k][0], dp[i-1][k][1] + prices[i])
dp[i][k][1] = max(dp[i-1][k][1], dp[i-1][k-1][0] - prices[i])
                ↑ 只有买入才算"开启一次交易"
```

## 总结

买卖股票 II 的核心：**在不限制交易次数时，能否获利完全不关心交易的位置，只关心能否低买高卖**。

四种方法层次递进：

1. **递归**——看见"决策树"的全貌
2. **DP**——把递归转化为可递推的状态转移表
3. **空间优化**——证明这道题只需要记住两个数
4. **贪心**——揭示出本质，抓住每一段上涨

面试中最理想的展示路径：先说"这本质是状态机"，写出 DP 状态转移方程，然后推出空间优化版，最后提一句"其实贪心也行"——展示从不同层次思考同一个问题的能力。

从递归到贪心，从 O(2^n) 的穷举到 O(1) 空间的双变量滚动——一个简单问题的不同解法，恰恰是算法思维最性感的体现。
