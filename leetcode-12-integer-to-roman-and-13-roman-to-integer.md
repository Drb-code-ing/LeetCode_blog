# LeetCode 12 & 13. 整数转罗马数字 & 罗马数字转整数 —— 从映射表到贪心，彻底吃透罗马数字

## 前言

罗马数字是编程面试中的经典字符串题，两道题互为逆操作：

- **罗马数字转整数（13）**：从左到右扫描，遇到"小前大"就减，否则加——一道 Easy 题，但有坑
- **整数转罗马数字（12）**：从大到小贪心匹配，不断减去最大的可用符号——Medium 难度，核心在于如何优雅处理 4、9 这类特殊情况

两道题放在一起写，因为它们共享同一套符号映射体系。理解了映射关系，两道题都能一通百通。

本文将为每道题提供多种解法，每种方法都附带 JavaScript 和 Python 双语言实现，以及详细的手动推演过程。

# 第一部分：罗马数字转整数（LeetCode 13）

## 问题描述

### LeetCode 13. Roman to Integer（罗马数字转整数）

> 罗马数字包含以下七种字符：`I`、`V`、`X`、`L`、`C`、`D`、`M`。
>
> ```
> 字符          数值
> I             1
> V             5
> X             10
> L             50
> C             100
> D             500
> M             1000
> ```
>
> 例如，罗马数字 `2` 写作 `II`，即两个 `1` 相加。`12` 写作 `XII`，即 `X + II`。`27` 写作 `XXVII`，即 `XX + V + II`。
>
> 通常情况下，罗马数字中小的数字在大的数字右边。但也存在特例，例如 `4` 不写作 `IIII`，而是 `IV`。数字 `1` 在数字 `5` 的左边，所表示的数等于大数减小数（`5 - 1 = 4`）。同样，数字 `9` 表示为 `IX`。
>
> 给定一个罗马数字，将其转换成整数。

**示例：**

```
输入：s = "III"
输出：3

输入：s = "IV"
输出：4

输入：s = "MCMXCIV"
输出：1994
解释：M=1000, CM=900, XC=90, IV=4
```

---

## 核心思想

**罗马数字的唯一规则：小数在大数左边表示减法，小数在大数右边表示加法。**

```
正常情况（加法）：VI = 5 + 1 = 6
特殊情况（减法）：IV = 5 - 1 = 4

判断方法：如果当前字符代表的值 < 下一个字符代表的值 → 减法
          否则 → 加法
```

记住这一个规则，整道题就解开了。

---

## 思路分析

### 解法一：一次遍历（最优解）

从左到右扫描每个字符，比较当前值和下一个值：

1. 如果当前值 < 下一个值 → 减去当前值（减法情况）
2. 否则 → 加上当前值（正常情况）

只需要一次遍历，O(n) 时间，O(1) 空间。

### 解法二：先替换再求和

先把所有"减法组合"替换成对应数值的负数表示，再逐字符求和：

```
"IV" → "4"   但这样替换后 "4" 是两位字符，需要特殊处理
```

这种方法不够优雅，但能帮助理解题意。

### 解法三：从右到左遍历

从右往左扫描，维护一个"当前最大值"。如果当前值 ≥ 最大值，加；否则减。思路从右往左更自然，但实现上不如解法一简洁。

---

## 代码实现

### JavaScript 版本

#### 方法一：一次遍历（最优解）

```javascript
/**
 * @param {string} s
 * @return {number}
 */
var romanToInt = function(s) {
    const map = { I: 1, V: 5, X: 10, L: 50, C: 100, D: 500, M: 1000 };
    let result = 0;

    for (let i = 0; i < s.length; i++) {
        const curr = map[s[i]];
        const next = map[s[i + 1]];  // 最后一位时 next 为 undefined

        // 当前值 < 下一个值 → 减法（如 IV 中的 I）
        if (next && curr < next) {
            result -= curr;
        } else {
            result += curr;
        }
    }

    return result;
};
```

#### 方法二：从右到左遍历

```javascript
var romanToInt = function(s) {
    const map = { I: 1, V: 5, X: 10, L: 50, C: 100, D: 500, M: 1000 };
    let result = 0;
    let maxVal = 0;  // 右边出现过的最大值

    // 从右往左扫描
    for (let i = s.length - 1; i >= 0; i--) {
        const val = map[s[i]];

        if (val >= maxVal) {
            // 当前值 ≥ 右边最大值 → 加法
            result += val;
            maxVal = val;
        } else {
            // 当前值 < 右边最大值 → 减法（如 IV 中的 I）
            result -= val;
        }
    }

    return result;
};
```

### Python 版本

#### 方法一：一次遍历（最优解）

```python
def roman_to_int(s: str) -> int:
    """方法一：一次遍历"""
    roman = {'I': 1, 'V': 5, 'X': 10, 'L': 50,
             'C': 100, 'D': 500, 'M': 1000}
    result = 0

    for i in range(len(s)):
        curr = roman[s[i]]
        # 如果当前值 < 下一个值 → 减法
        if i + 1 < len(s) and curr < roman[s[i + 1]]:
            result -= curr
        else:
            result += curr

    return result
```

#### 方法二：从右到左遍历

```python
def roman_to_int(s: str) -> int:
    """方法二：从右到左遍历"""
    roman = {'I': 1, 'V': 5, 'X': 10, 'L': 50,
             'C': 100, 'D': 500, 'M': 1000}
    result = 0
    max_val = 0  # 右边出现过的最大值

    for ch in reversed(s):
        val = roman[ch]
        if val >= max_val:
            result += val
            max_val = val
        else:
            result -= val

    return result
```

# 第二部分：整数转罗马数字（LeetCode 12）

## 问题描述

### LeetCode 12. Integer to Roman（整数转罗马数字）

> 给定一个整数，将其转为罗马数字。
>
> 输入范围：`1 <= num <= 3999`

**示例：**

```
输入：num = 3
输出："III"

输入：num = 4
输出："IV"

输入：num = 1994
输出："MCMXCIV"
解释：M=1000, CM=900, XC=90, IV=4
```

---

## 核心思想

**整数转罗马数字的本质是"贪心匹配"：每次从最大的符号开始，能减多少减多少。**

但罗马数字有一个特殊规则：**4 和 9 不用重复符号表示，而是用"减法形式"**：

```
4  → IV（不是 IIII）
9  → IX（不是 VIIII）
40 → XL（不是 XXXX）
90 → XC（不是 LXXXX）
400 → CD（不是 CCCC）
900 → CM（不是 DCCCC）
```

所以，映射表里不仅要有基本符号（1、5、10...），还要包含这些减法形式（4、9、40...）。

**把 13 个符号从小到大列出来，从大到小贪心匹配，就是最优解。**

---

## 思路分析

### 解法一：贪心 + 映射表（最优解）

构建一个包含 13 个符号的有序映射表（从大到小）：

```
1000 → M
900  → CM
500  → D
400  → CD
100  → C
90   → XC
50   → L
40   → XL
10   → X
9    → IX
5    → V
4    → IV
1    → I
```

从最大的值开始，每次能减就减，拼接对应符号。

### 解法二：按位处理

把整数按千位、百位、十位、个位拆开，每一位单独转换：

```
1994 → 千位(1) + 百位(9) + 十位(9) + 个位(4)
     → M + CM + XC + IV
     → "MCMXCIV"
```

每一位只有 10 种可能（0-9），可以用查表法。

### 解法三：暴力枚举

硬编码每一位的所有可能：个位 1-9 对应 I 到 IX，十位 10-90 对应 X 到 XC... 拼接即可。代码冗长但逻辑最简单。

---

## 代码实现

### JavaScript 版本

#### 方法一：贪心 + 映射表（最优解）

```javascript
/**
 * @param {number} num
 * @return {string}
 */
var intToRoman = function(num) {
    // 从大到小排列的值-符号对，包含减法形式
    const values = [1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1];
    const symbols = ['M', 'CM', 'D', 'CD', 'C', 'XC', 'L', 'XL', 'X', 'IX', 'V', 'IV', 'I'];

    let result = '';

    for (let i = 0; i < values.length; i++) {
        // 当前值能减几次就减几次
        while (num >= values[i]) {
            result += symbols[i];
            num -= values[i];
        }
    }

    return result;
};
```

#### 方法二：按位处理

```javascript
var intToRoman = function(num) {
    // 每一位的 10 种情况
    const ones = ['', 'I', 'II', 'III', 'IV', 'V', 'VI', 'VII', 'VIII', 'IX'];
    const tens = ['', 'X', 'XX', 'XXX', 'XL', 'L', 'LX', 'LXX', 'LXXX', 'XC'];
    const hundreds = ['', 'C', 'CC', 'CCC', 'CD', 'D', 'DC', 'DCC', 'DCCC', 'CM'];
    const thousands = ['', 'M', 'MM', 'MMM'];

    return thousands[Math.floor(num / 1000)] +
           hundreds[Math.floor(num % 1000 / 100)] +
           tens[Math.floor(num % 100 / 10)] +
           ones[num % 10];
};
```

### Python 版本

#### 方法一：贪心 + 映射表（最优解）

```python
def int_to_roman(num: int) -> str:
    """方法一：贪心 + 映射表"""
    # 从大到小排列的值-符号对，包含减法形式
    val_sym = [
        (1000, 'M'), (900, 'CM'), (500, 'D'), (400, 'CD'),
        (100, 'C'),  (90, 'XC'),  (50, 'L'),  (40, 'XL'),
        (10, 'X'),   (9, 'IX'),   (5, 'V'),   (4, 'IV'),
        (1, 'I')
    ]

    result = []
    for val, sym in val_sym:
        while num >= val:
            result.append(sym)
            num -= val

    return ''.join(result)
```

#### 方法二：按位处理

```python
def int_to_roman(num: int) -> str:
    """方法二：按位处理"""
    ones = ['', 'I', 'II', 'III', 'IV', 'V', 'VI', 'VII', 'VIII', 'IX']
    tens = ['', 'X', 'XX', 'XXX', 'XL', 'L', 'LX', 'LXX', 'LXXX', 'XC']
    hundreds = ['', 'C', 'CC', 'CCC', 'CD', 'D', 'DC', 'DCC', 'DCCC', 'CM']
    thousands = ['', 'M', 'MM', 'MMM']

    return (thousands[num // 1000] +
            hundreds[num % 1000 // 100] +
            tens[num % 100 // 10] +
            ones[num % 10])
```

# 第三部分：两题对比

## 为什么放在一起？

| 维度 | 罗马数字转整数（13） | 整数转罗马数字（12） |
|------|-------------------|-------------------|
| **方向** | 字符串 → 数字 | 数字 → 字符串 |
| **核心规则** | 小前大则减，否则加 | 贪心匹配，能减就减 |
| **关键数据结构** | 哈希表（单字符映射） | 有序数组（含减法形式） |
| **时间复杂度** | O(n)，n 为字符串长度 | O(1)，数字上限固定 |
| **共同点** | 都依赖同一套罗马数字符号体系 |

### 两题的核心联系

```
罗马数字转整数：扫描字符串，用 map 做 O(1) 查找
整数转罗马数字：贪心匹配，用有序数组做 O(1) 查找

两题共享的核心知识：
  7 个基本符号: I=1, V=5, X=10, L=50, C=100, D=500, M=1000
  6 个减法形式: IV=4, IX=9, XL=40, XC=90, CD=400, CM=900
```

---

## 优化方向

### 1. 罗马数字转整数的边界情况

最常见的坑：

```
输入 "I"  → 只有一个字符，next 为 undefined，直接加 1
输入 "IV" → I < V，减法，结果 4
输入 "III" → 全部加法，结果 3
```

解法一中 `next` 可能为 `undefined`（最后一个字符），JavaScript 中 `undefined` 参与比较会得到 `false`，所以 `curr < next` 自动为 `false`，走加法分支。Python 中需要显式判断 `i + 1 < len(s)`。

### 2. 整数转罗马数字的映射表设计

为什么贪心法的映射表必须包含减法形式（4、9、40...）？

```
如果不包含 900→CM:
  900 = 500 + 100 + 100 + 100 + 100 = "DCCCC"
  但正确答案是 "CM"

如果不包含 4→IV:
  4 = 1 + 1 + 1 + 1 = "IIII"
  但正确答案是 "IV"
```

**减法形式保证了每个数字只有一种标准表示。** 这也是为什么映射表恰好有 13 个条目（7 个基本 + 6 个减法形式）。

### 3. 从罗马数字扩展

如果输入范围扩大到 4000 以上，需要引入新的符号。历史上罗马数字没有标准的千位以上表示，常见的扩展方式是加横线表示 ×1000（如 `V̅` = 5000）。编程中可以简单地在 M 前面加括号或下划线标记。

## 总结

1. **罗马数字转整数的核心是"相邻比较"**：只要记住"小前大则减"这一条规则，一次遍历就能解决。两种方向的遍历（左→右 / 右→左）都可以，左→右更直观。

2. **整数转罗马数字的核心是"贪心 + 完整映射表"**：映射表必须包含 13 个条目（7 基本 + 6 减法形式），从大到小匹配能减就减。按位处理是另一种优雅思路，但需要硬编码每一位的所有情况。

3. **两道题共享同一套符号体系**：理解了 7 个基本符号和 6 个减法形式之间的关系，两道题都能迎刃而解。

面试中建议这样展示思路：先说清楚罗马数字的规则（加法 vs 减法），再画出映射表，最后给出代码。面试官看重的不是你能不能写出来，而是你对规则的理解是否清晰。

建议手动推演几个用例：`III`（全加法）、`IV`（减法）、`MCMXCIV`（混合），确保对每种情况都能正确处理。
