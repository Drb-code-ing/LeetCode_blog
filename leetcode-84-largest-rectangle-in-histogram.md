## 前言

如果说栈是算法世界的瑞士军刀，那**单调栈（Monotonic Stack）**就是这把刀上最锋利的刃口——专门处理一类问题：**在数组中快速找到每个元素左边（或右边）第一个比它大（或小）的元素**。

**柱状图中最大的矩形**（LeetCode 84）是单调栈的"封神之作"。这道 Hard 题看起来平平无奇——给一个直方图，找出能勾勒出的最大矩形面积——但背后却藏着一个精妙的线性解法。不理解单调栈的人会用 O(n²) 暴力碰运气，理解的人用 O(n) 优雅秒杀。

本文将带你从暴力解出发，一步步推导出单调栈的 O(n) 解法，让你彻底搞懂"为什么栈能算出最大矩形"。

## 单调栈核心思想

### 什么是单调栈？

单调栈就是**栈内元素保持单调递增或单调递减**的栈。

- **单调递增栈**：栈顶到栈底递增（即栈顶最小）
- **单调递减栈**：栈顶到栈底递减（即栈顶最大）

### 它能解决什么问题？

一句话：**快速找到数组中每个元素左边/右边第一个比它小/大的元素**。

以 `[2,1,5,6,2,3]` 为例，"右边第一个比它小的元素"：

| 元素 | 右边第一个比它小的 | 距离多远？ |
|------|-------------------|-----------|
| 2 (索引0) | 1 (索引1) | 距离 1 |
| 1 (索引1) | ❌ 没有 | 到末尾 |
| 5 (索引2) | 2 (索引4) | 距离 2 |
| 6 (索引3) | 2 (索引4) | 距离 1 |
| 2 (索引4) | ❌ 没有 | 到末尾 |
| 3 (索引5) | ❌ 没有 | 到末尾 |

用暴力法需要 O(n²)，而单调栈只需要 **O(n)**。

### 单调栈模板

```javascript
// 单调递增栈模板：找右边第一个更小的元素
function nextSmaller(arr) {
    const stack = [];
    const result = new Array(arr.length).fill(-1);

    for (let i = 0; i < arr.length; i++) {
        // 当前元素比栈顶小，说明栈顶遇到了右边第一个更小值
        while (stack.length > 0 && arr[i] < arr[stack[stack.length - 1]]) {
            const idx = stack.pop();
            result[idx] = i;  // 右边第一个更小元素的索引
        }
        stack.push(i);
    }

    return result;
}
```

关键点：**栈中存的是索引，不存值**。通过索引可以随时访问值，同时还能知道位置。

## 思路分析：柱状图中最大的矩形

### 核心观察

对于每一根柱子，以它的高度作为矩形高度时，矩形能往左右延伸多远？

**答案是：直到遇到比它矮的柱子为止。**

```
对于 heights[2] = 5（索引 2）：
  往左找：索引 1 的高度是 1 < 5，停 → 左边界 = 2
  往右找：索引 4 的高度是 2 < 5，停 → 右边界 = 3
  宽度 = 3 - 2 + 1 = 2（只有索引 2 和 3）
  面积 = 5 × 2 = 10 ← 最大！

对于 heights[1] = 1（索引 1）：
  往左找：没有更矮的 → 左边界 = 0
  往右找：没有更矮的 → 右边界 = 5
  宽度 = 5 - 0 + 1 = 6
  面积 = 1 × 6 = 6
```

所以解题关键是：**对于每个柱子，找到它左边第一个更矮的柱子和右边第一个更矮的柱子**——这正是单调栈的用武之地。

### 算法流程

1. 用单调递增栈（栈顶到栈底递增）来维护柱子的索引
2. 遍历每个柱子：
   - 如果当前柱子高度 < 栈顶柱子高度 → 栈顶遇到了右边第一个更矮的 → 出栈计算面积
   - 否则 → 入栈
3. 遍历结束后，栈中剩余的柱子右边没有更矮的，右边界为数组末尾

计算面积时需要的参数：
- **高度** = `heights[poppedIndex]`
- **右边界** = 当前索引 `i`（右边第一个更矮的柱子位置）
- **左边界** = 新栈顶（左边第一个更矮的柱子位置），如果栈为空则左边界为 -1
- **宽度** = `右边界 - 左边界 - 1`

## 代码实现

### JavaScript 版本（单调栈）

```javascript
/**
 * @param {number[]} heights
 * @return {number}
 */
var largestRectangleArea = function(heights) {
    const stack = [];      // 单调递增栈，存储索引
    let maxArea = 0;

    // 在末尾加一个高度 0 的哨兵，强制清空栈
    const h = [...heights, 0];

    for (let i = 0; i < h.length; i++) {
        // 当前高度小于栈顶高度 → 栈顶无法继续向右扩展，出栈结算
        while (stack.length > 0 && h[i] < h[stack[stack.length - 1]]) {
            const height = h[stack.pop()];       // 出栈柱子的高度
            const rightBoundary = i;              // 右边第一个更矮的柱子
            const leftBoundary = stack.length === 0
                ? -1
                : stack[stack.length - 1];        // 左边第一个更矮的柱子
            const width = rightBoundary - leftBoundary - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }

    return maxArea;
};
```

### 逐步推演

以 `heights = [2,1,5,6,2,3]` + 哨兵 `0` 为例：

```
初始：stack = [], maxArea = 0

i=0, h[0]=2 → stack=[0]
i=1, h[1]=1 < h[0]=2 → 弹出 0：
      高度=2, 左边界=-1, 右边界=1, 宽度=2, 面积=4, maxArea=4
    → stack=[], 然后 push(1) → stack=[1]

i=2, h[2]=5 > h[1]=1 → push(2) → stack=[1,2]
i=3, h[3]=6 > h[2]=5 → push(3) → stack=[1,2,3]

i=4, h[4]=2 < h[3]=6 → 弹出 3：
      高度=6, 左边界=2, 右边界=4, 宽度=1, 面积=6, maxArea=6
    → h[4]=2 < h[2]=5 → 弹出 2：
      高度=5, 左边界=1, 右边界=4, 宽度=2, 面积=10, maxArea=10
    → h[4]=2 > h[1]=1 → push(4) → stack=[1,4]

i=5, h[5]=3 > h[4]=2 → push(5) → stack=[1,4,5]

i=6, h[6]=0 < h[5]=3 → 弹出 5：
      高度=3, 左边界=4, 右边界=6, 宽度=1, 面积=3
    → h[6]=0 < h[4]=2 → 弹出 4：
      高度=2, 左边界=1, 右边界=6, 宽度=4, 面积=8
    → h[6]=0 < h[1]=1 → 弹出 1：
      高度=1, 左边界=-1, 右边界=6, 宽度=7, 面积=7

最终 maxArea = 10 ✅
```

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n) | 每个元素入栈一次、出栈最多一次 |
| 空间复杂度 | O(n) | 最坏情况下（单调递增）栈中存储所有 n 个元素的索引 |

每个柱子只会入栈一次、出栈一次，所以虽然是双重循环（for 里面套 while），但总操作次数是 O(n)——这就是单调栈优雅的地方。

## 优化方向

### 1. 暴力法（O(n²)，了解即可）

```javascript
var largestRectangleArea = function(heights) {
    let maxArea = 0;
    for (let i = 0; i < heights.length; i++) {
        let left = i, right = i;
        while (left >= 0 && heights[left] >= heights[i]) left--;
        while (right < heights.length && heights[right] >= heights[i]) right++;
        maxArea = Math.max(maxArea, heights[i] * (right - left - 1));
    }
    return maxArea;
};
```

每个柱子向左右扩展到边界，最坏情况 O(n²)。面试时可以作为起点，然后优化到单调栈。

### 2. 两次遍历求左右边界

先用一次单调栈算出每个柱子的「左边第一个更小」和「右边第一个更小」，存到数组中，然后再遍历一次计算面积。思路更清晰，但需要额外空间：

```javascript
var largestRectangleArea = function(heights) {
    const n = heights.length;
    const left = new Array(n).fill(-1);   // 左边第一个更小的索引
    const right = new Array(n).fill(n);   // 右边第一个更小的索引

    // 求左边第一个更小
    const stack = [];
    for (let i = 0; i < n; i++) {
        while (stack.length > 0 && heights[i] <= heights[stack[stack.length - 1]]) {
            stack.pop();
        }
        left[i] = stack.length === 0 ? -1 : stack[stack.length - 1];
        stack.push(i);
    }

    // 清空栈，求右边第一个更小
    stack.length = 0;
    for (let i = n - 1; i >= 0; i--) {
        while (stack.length > 0 && heights[i] <= heights[stack[stack.length - 1]]) {
            stack.pop();
        }
        right[i] = stack.length === 0 ? n : stack[stack.length - 1];
        stack.push(i);
    }

    // 计算最大面积
    let maxArea = 0;
    for (let i = 0; i < n; i++) {
        maxArea = Math.max(maxArea, heights[i] * (right[i] - left[i] - 1));
    }
    return maxArea;
};
```

### 3. 哨兵技巧

在原始数组末尾添加一个高度为 0 的哨兵，保证遍历结束后栈被完全清空，避免遍历结束后还需要单独处理栈中剩余元素。

## 总结

柱状图中最大矩形这道题有两个核心洞察：

1. **以每个柱子为高，向左右扩展到第一个更矮的柱子**——这是解决问题的几何直觉
2. **单调栈能在 O(n) 时间内找到每个元素的左右边界**——这是实现高效的算法工具

两个版本中，**一次遍历 + 哨兵**的实现最简洁优雅，也最推荐在面试中使用。如果担心不好理解，可以先写"两次遍历求左右边界"的版本，思路更线性，再优化成一次遍历。

最后记住一句话：**单调栈不存值，存索引；不比较大小，比较位置。** 索引包含了值和位置双重信息，这就是它强大的原因。

建议在纸上推演一遍 `[2,1,5,6,2,3]` 加上哨兵后的完整运行过程——每一步都写下 `stack` 和 `maxArea` 的变化。理解清楚之后，你会发现自己看所有"找左右边界"的题目都通透了许多。
